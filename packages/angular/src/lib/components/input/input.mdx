# Input

The Input component provides a clean, accessible text input field with built-in validation support and consistent styling. It implements the `FormValueControl<string>` interface from Angular's signals-based forms, providing seamless integration with modern reactive forms while maintaining backwards compatibility.

## Usage

```typescript
import { UniInputComponent } from '@uni-design-system/uni-angular';
```

## Methods

- `markAsTouched()`: Manually marks the input as touched, triggering error display if validation fails

## Basic Usage with Signals

```html
<uni-input
  label="Email Address"
  placeholder="Enter your email"
  [(value)]="emailValue"
  [(touched)]="emailTouched"
  [invalid]="emailInvalid"
  [dirty]="emailDirty"
>
</uni-input>
```

```typescript
import { Component, signal } from '@angular/core';

export class MyComponent {
  // Using Angular signals for reactive state management
  emailValue = signal('');
  emailTouched = signal(false);
  emailInvalid = signal(false);
  emailDirty = signal(false);

  validateEmail() {
    const hasAtSymbol = this.emailValue().includes('@');
    this.emailInvalid.set(!hasAtSymbol);
  }
}
```

## Signals-Based Form Integration

The component integrates seamlessly with Angular's new signals-based forms and the `FormValueControl` interface:

```typescript
import { Component } from '@angular/core';
import { FormControl } from '@angular/forms/signals';

export class MyFormComponent {
  // Create a signals-based form control
  emailControl = new FormControl('', {
    validators: [
      /* your validators */
    ],
  });

  onSubmit() {
    if (this.emailControl.valid()) {
      console.log('Email:', this.emailControl.value());
    }
  }
}
```

```html
<!-- Direct binding with FormControl signals -->
<uni-input
  label="Email Address"
  placeholder="Enter your email"
  [(value)]="emailControl.value"
  [(touched)]="emailControl.touched"
  [invalid]="emailControl.invalid"
  [dirty]="emailControl.dirty"
  [disabled]="emailControl.disabled"
>
</uni-input>
```

## Error Display

Validation errors are only shown when the field is `invalid` and the user has interacted with it (either `touched` or `dirty`), providing optimal user experience by not showing premature validation errors.

```typescript
// Error display logic (computed automatically in the component)
protected readonly showError = computed(
  () => this.invalid() && (this.touched() || this.dirty())
);
```

The error display logic considers both `touched` (user has focused/blurred the field) and `dirty` (user has changed the value) states to determine when to show validation errors, ensuring users aren't confronted with errors before they've had a chance to interact with the field.

## Styling

The component integrates with the theme system through the `UniInputBoxComponent` wrapper and provides consistent styling across platforms. Error styling is automatically applied when validation fails and the user has interacted with the field. The component uses Emotion CSS-in-JS for styling and supports theme switching through the theme service.

## Implementation Details

### FormValueControl Compliance

- **Interface**: Implements `FormValueControl<string>` from `@angular/forms/signals`
- **Required Signals**: `value`, `disabled`, `touched`, `invalid`, `dirty`
- **Methods**: `markAsTouched()` for programmatic interaction control
- **Computed Properties**: `showError` for intelligent error display logic

### Signal Architecture

- Uses Angular's modern signal-based reactivity system
- `model()` signals for two-way data binding (`value`, `touched`)
- `input()` signals for one-way configuration and state (`disabled`, `invalid`, `dirty`)
- `computed()` signals for derived state (`showError`)

### Template Integration

- Clean template with minimal logic
- Proper event handling for user interactions
- Content projection support for extensible UI patterns
