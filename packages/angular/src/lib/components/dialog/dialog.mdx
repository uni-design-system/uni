## Overview

Enhanced modal dialog that extends the native HTML `<dialog>` element with built-in animations, composition components, and accessibility features.

### Features

- **Native HTML Dialog**: Built on the standard dialog element with full semantic support
- **Smooth Animations**: Automatic fade transitions for open/close states
- **Composition Components**: Optional `DialogHeader` and `DialogButtons` for consistent layouts
- **Programmatic Control**: Open/close methods plus show property binding
- **Backdrop Handling**: Click-outside-to-close with customizable behavior
- **Accessibility**: Full keyboard navigation, focus trapping, and screen reader support

### Basic Usage

```html
<dialog #dialog>
  <DialogHeader>Dialog Title</DialogHeader>
  <div>Dialog content goes here</div>
  <DialogButtons>
    <button (click)="dialog.close()">Close</button>
  </DialogButtons>
</dialog>

<button (click)="dialog.open()">Open Dialog</button>
```

### Advanced Configurations

#### With Custom Close Button

```html
<dialog #dialog [defaultCloseButton]="false">
  <DialogHeader>
    Custom Header
    <button
      variant="ghost"
      (click)="dialog.close()"
      style="position: absolute; right: 8px; top: 8px;"
    >
      Close
    </button>
  </DialogHeader>
  <div>Dialog content with custom close button</div>
</dialog>
```

#### Binding to Show Property

```html
<dialog [show]="isDialogVisible" (showing)="dialogStateChanged($event)">
  <DialogHeader>Controlled Dialog</DialogHeader>
  <div>This dialog is controlled by the isDialogVisible property</div>
  <DialogButtons>
    <button (click)="isDialogVisible = false">Close</button>
  </DialogButtons>
</dialog>

<button (click)="isDialogVisible = true">Open Dialog</button>
```

## Best Practices

**Use for:** Important notifications, confirmations, user input forms
**Avoid:** Non-critical information, nested dialogs, complex navigation

## Accessibility

Built on the native `<dialog>` element for automatic ARIA support. Includes:

- Focus trapping and keyboard navigation (Tab, Escape, Enter)
- Screen reader announcements
- Semantic dialog roles
